## Gotchas

- **Redis-only.** Don't stand up Postgres for metrics. If exp5 wants a "real backend," a `time.sleep(2ms)` is a fine stand-in.
- **Drive all window math from ONE monotonic clock in the harness.** The post's "wobbling reset" bug is precisely about mixing Redis TTL with the app clock — the harness must not reproduce that accidentally; compute windows from a single `time.time()`/`time.monotonic()` source and store absolute reset timestamps (as the atomic Lua does).
- **exp5 load-gen bottleneck:** threads won't saturate Redis (GIL). Use multiprocessing + pipelined `EVALSHA`, or `redis-benchmark`. Otherwise you're benchmarking Python and the "single-thread ceiling" number is meaningless.
- **exp3 replica lag:** localhost replication is ~instant. Use toxiproxy (or accept a low rate and say so). Connect **decision reads to the replica port, increments to the primary port** — that split is the whole point.
- **exp2 ground truth:** count admits with an AtomicCounter inside the limiter's allow-path, and compare to `limit × windows_elapsed`; don't try to infer overshoot from Redis state after the fact.
- **Dark-mode charts:** test BOTH themes via the site's `data-theme` toggle, not just the OS media query.
